The single verification endpoint lets you verify one email address in real-time. Each request performs syntax, domain, MX record, and SMTP checks and typically responds in under 500ms. You receive more than 15 data points per result — enough to make confident send/no-send decisions in your application.

## Endpoint

```
POST https://mailvalid.io/api/v1/verify/single
```

<Note>
  **Base URL:** `https://mailvalid.io/api/v1`
</Note>

## Request

<ParamField body="email" type="string" required>
  The email address to verify.
</ParamField>

<ParamField header="X-API-Key" type="string" required>
  Your API key. Generate one from the [Dashboard → API Keys](https://mailvalid.io/dashboard/api-keys).
</ParamField>

<CodeGroup>

```bash cURL
curl -X POST https://mailvalid.io/api/v1/verify/single \
  -H "X-API-Key: mv_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"email": "john@company.com"}'
```

```python Python
import requests

API_KEY = "mv_live_your_key_here"

response = requests.post(
    "https://mailvalid.io/api/v1/verify/single",
    headers={"X-API-Key": API_KEY},
    json={"email": "john@company.com"}
)

result = response.json()
print(f"Status: {result['result']['status']}")  # valid, invalid, catch_all, unknown, do_not_mail
print(f"Valid: {result['result']['is_valid']}")
print(f"Confidence: {result['result']['confidence_score']}")
print(f"Reason: {result['result']['status_reason']}")
```

```javascript JavaScript
const API_KEY = "mv_live_your_key_here";

const response = await fetch(
  "https://mailvalid.io/api/v1/verify/single",
  {
    method: "POST",
    headers: {
      "X-API-Key": API_KEY,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ email: "john@company.com" })
  }
);

const data = await response.json();
console.log(`Status: ${data.result.status}`);  // valid, invalid, catch_all, unknown, do_not_mail
console.log(`Valid: ${data.result.is_valid}`);
console.log(`Confidence: ${data.result.confidence_score}`);
```

</CodeGroup>

## Response

A successful request returns a JSON object with a top-level `result` containing the full verification data:

```json
{
  "success": true,
  "credits_used": 1,
  "result": {
    "email": "john@company.com",
    "status": "valid",
    "is_valid": true,
    "syntax_valid": true,
    "domain": "company.com",
    "domain_valid": true,
    "has_mx": true,
    "mx_records": [
      {"priority": 10, "host": "mx1.company.com"}
    ],
    "smtp_checked": true,
    "smtp_response_code": 250,
    "is_disposable": false,
    "is_role_based": false,
    "is_catch_all": false,
    "is_free_provider": false,
    "confidence_score": 95,
    "status_reason": "mailbox_confirmed",
    "provider": "corporate",
    "verification_time_ms": 234,
    "cached": false
  }
}
```

### Response fields

| Field | Type | Description |
|---|---|---|
| `success` | boolean | Whether the API call succeeded |
| `credits_used` | integer | Credits consumed (0 or 1) |
| `result.email` | string | The verified email address |
| `result.status` | string | `valid`, `invalid`, `catch_all`, `unknown`, `do_not_mail` |
| `result.is_valid` | boolean | `true` means safe to send |
| `result.syntax_valid` | boolean | Email format is RFC 5322 compliant |
| `result.domain` | string | Domain part of the email |
| `result.domain_valid` | boolean | Domain exists in DNS |
| `result.has_mx` | boolean | Domain has MX records |
| `result.mx_records` | array | List of MX records with priority and host |
| `result.smtp_checked` | boolean | SMTP verification was performed |
| `result.smtp_response_code` | integer | SMTP response code (250 = accepted) |
| `result.is_disposable` | boolean | Temporary/throwaway email provider |
| `result.is_role_based` | boolean | Role-based address (info@, support@, etc.) |
| `result.is_catch_all` | boolean | Domain accepts all addresses |
| `result.is_free_provider` | boolean | Free email provider (Gmail, Yahoo, etc.) |
| `result.confidence_score` | integer | 0–100 confidence in the result |
| `result.status_reason` | string | Machine-readable sub-status |
| `result.provider` | string | Detected email provider |
| `result.verification_time_ms` | integer | Time taken in milliseconds |
| `result.cached` | boolean | `true` = served from cache, 0 credits used |

<Tip>
  Use `is_valid: true` **and** `is_disposable: false` as your primary filter for safe sends. This combination screens out both undeliverable addresses and throwaway inboxes in a single check.
</Tip>

For the full list of `status` and `status_reason` values with recommended actions, see [Email Statuses](/concepts/email-statuses).
